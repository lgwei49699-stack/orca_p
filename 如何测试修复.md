# 如何测试 GLEW 修复

## 🚀 快速开始（在 Linux 服务器上）

### 1. 拉取最新代码
```bash
git pull
# 或者如果您修改了本地文件
git stash
git pull
git stash pop
```

### 2. 重新编译
```bash
# 清理旧的编译
rm -rf build

# 创建新的编译目录
mkdir build
cd build

# 配置项目
cmake -DCMAKE_BUILD_TYPE=Release ..

# 编译（使用所有 CPU 核心）
cmake --build . --target OrcaSlicer --parallel $(nproc)
```

### 3. 运行测试
```bash
cd ..  # 返回项目根目录

# 使用测试脚本
./test_thumbnail_fix.sh your_model.3mf

# 或者手动运行
xvfb-run -a ./build/src/OrcaSlicer --export-gcode input.3mf --output output.gcode
```

## ✅ 如何判断修复成功

### 看日志输出
成功的日志应该包含：
```
[info] ✓ GLEW initialized successfully!
[info] ✓ OpenGL Context Information:
[info]   Version:  2.1 Mesa ... (或 3.3 Mesa ...)
[info]   Renderer: llvmpipe
[info] ✓✓✓ OpenGL Setup Complete!
```

### 检查输出文件
```bash
# 查看是否生成了 gcode 文件
ls -lh output.gcode

# 检查是否包含缩略图
grep "thumbnail begin" output.gcode
```

如果看到类似这样的输出，说明成功了：
```
; thumbnail begin 300x300 12345
; iVBORw0KGgoAAAANSUhEUgAA...
; thumbnail end
```

## 🐛 如果还是失败

### 检查系统环境
```bash
# 1. 检查是否安装了必要的包
dpkg -l | grep -E "xvfb|mesa|libgl"

# 2. 检查 Mesa 是否工作
LIBGL_ALWAYS_SOFTWARE=1 glxinfo | grep "OpenGL version"

# 3. 如果 glxinfo 不存在
sudo apt-get install -y mesa-utils
```

### 查看完整日志
日志会输出到终端，请完整复制并检查：
1. 查找 `[error]` 行 - 这是错误信息
2. 查找 `GLEW initialization FAILED` - GLEW 初始化失败
3. 查找 `Attempting fallback` - 是否尝试了降级

### 常见问题

**问题 1**: `Unable to init glew library` + `Missing GL version`
```
原因：Mesa 不支持 OpenGL 3.3
解决：代码会自动降级到 OpenGL 2.1，查看是否出现 "Attempting fallback" 日志
```

**问题 2**: `Failed to create GLFW window`
```
原因：DISPLAY 环境变量未设置或 Xvfb 未运行
解决：确保使用 xvfb-run，或手动启动 Xvfb
```

**问题 3**: 段错误（Segmentation fault）
```
原因：可能是编译不完整或依赖库版本不匹配
解决：完全清理并重新编译
    rm -rf build
    mkdir build
    cd build
    cmake ..
    cmake --build . --target OrcaSlicer --parallel $(nproc)
```

**问题 4**: 编译错误
```
原因：可能缺少依赖或语法错误
解决：查看编译输出，确保所有依赖已安装
```

3. **完整的运行日志**：

4. **编译信息**：
编译时是否有警告或错误

## 🎯 关键修复点

这次修复主要解决了三个问题：

### 1. **变量作用域错误**（最严重）
- **问题**：`bg_window` 变量在错误的作用域中使用
- **后果**：程序崩溃或未定义行为
- **修复**：在正确的作用域声明变量

### 2. **GLEW 重复初始化**
- **问题**：GLEW 只能初始化一次，第二次会失败
- **后果**：即使降级到 OpenGL 2.1 也无法成功
- **修复**：完整重置窗口和上下文

### 3. **Mesa OpenGL 3.3 支持不完整**
- **问题**：Mesa llvmpipe 在无头环境下 OpenGL 3.3 不完整
- **后果**：GLEW 无法获取 OpenGL 版本
- **修复**：自动降级到 OpenGL 2.1
