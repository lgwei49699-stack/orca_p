根本原因：
- GLX 尝试使用硬件 GPU 渲染
- 服务器环境没有 GPU，且没有配置 Mesa 软件渲染器
- 导致 OpenGL 上下文创建失败

### 最终解决方案（最简单可靠）

**设置 `LIBGL_ALWAYS_SOFTWARE=1` 强制 Mesa 使用 llvmpipe 软件渲染器**

这个方案：
- ✅ 改动最小（只需一行代码）
- ✅ 继续使用 GLX（不需要切换到 OSMesa）
- ✅ 利用成熟的 Mesa llvmpipe 软件渲染器
- ✅ 性能足够（llvmpipe 使用 CPU SIMD 优化）

### 代码修改

**src/OrcaSlicer.cpp（第 5948-5962 行）**

```cpp
// 设置环境变量强制使用 Mesa 软件渲染
setenv("LIBGL_ALWAYS_SOFTWARE", "1", 1);
```

**src/OrcaSlicer.cpp（第 6001-6004 行）**

```cpp
// 使用 GLX（配合 Mesa llvmpipe 软件渲染器）
glfwWindowHint(GLFW_CONTEXT_CREATION_API, GLFW_NATIVE_CONTEXT_API);
```
