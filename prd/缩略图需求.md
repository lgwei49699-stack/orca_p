## 2026-01-14 晚 - GLEW "Missing GL version" 最终解决方案

### 问题现象
即使设置了 `LIBGL_ALWAYS_SOFTWARE=1` 和 `glewExperimental = GL_TRUE`，GLEW 初始化仍然失败：
- GLFW 窗口创建成功
- OpenGL 上下文设置为当前成功
- **但 GLEW 报告 "Missing GL version"（错误代码 1）**

### 根本原因
**Mesa llvmpipe 软件渲染器不完全支持 OpenGL 3.3 Core Profile**，导致：
1. GLFW 可以创建 3.3 上下文，但这个上下文不完整
2. GLEW 在尝试查询 OpenGL 版本时失败（`glGetString(GL_VERSION)` 返回 NULL）
3. 即使设置 `glewExperimental` 也无法解决这个问题

### 解决方案：降级 + 诊断 + 自动降级

#### 1. Pre-GLEW 诊断
```cpp
// 在 glewInit 之前尝试获取 OpenGL 版本
typedef const GLubyte* (*PFNGLGETSTRINGPROC)(GLenum);
PFNGLGETSTRINGPROC glGetStringPtr = (PFNGLGETSTRINGPROC)glfwGetProcAddress("glGetString");
if (glGetStringPtr) {
    const char* pre_glew_version = (const char*)glGetStringPtr(GL_VERSION);
    if (pre_glew_version) {
        BOOST_LOG_TRIVIAL(info) << "Pre-GLEW GL_VERSION: " << pre_glew_version;
    } else {
        BOOST_LOG_TRIVIAL(warning) << "Pre-GLEW glGetString(GL_VERSION) returned NULL";
    }
}
```

#### 2. 自动降级到 OpenGL 2.1
如果 OpenGL 3.3 + GLEW 初始化失败：
```cpp
// Destroy 3.3 window
glfwDestroyWindow(bg_window);

// Recreate with OpenGL 2.1
glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 2);
glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 1);
glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_ANY_PROFILE);

bg_window = glfwCreateWindow(640, 480, "", nullptr, nullptr);
glfwMakeContextCurrent(bg_window);

// Retry GLEW init
```

#### 3. 增强的错误诊断
```cpp
BOOST_LOG_TRIVIAL(error) << "Common causes on Linux:";
BOOST_LOG_TRIVIAL(error) << "  1. OpenGL context version too high (try 2.1 instead of 3.3)";
BOOST_LOG_TRIVIAL(error) << "  2. Mesa llvmpipe doesn't support requested OpenGL features";
BOOST_LOG_TRIVIAL(error) << "  3. Missing or incompatible OpenGL libraries";
BOOST_LOG_TRIVIAL(error) << "  4. Invalid DISPLAY or X server configuration";
```

### 为什么 OpenGL 2.1 更可靠？
1. **Mesa llvmpipe 对老版本 OpenGL 支持更完整**
2. **不需要 Core Profile**，使用 Compatibility Profile
3. **所有缩略图渲染功能在 OpenGL 2.1 中都可用**
4. **GLEW 可以正确查询 OpenGL 2.1 的版本字符串**
